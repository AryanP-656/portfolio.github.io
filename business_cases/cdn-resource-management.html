Dynamic CDN Resource Management

Imagine it's the weekend, you're tired from a stressful week and ready to relax with some entertainment. You open Netflix and begin your journey….



Of course, you're not alone and there might be several others in your region and the neighbouring regions who had the same idea. Hence, Netflix must dynamically decide where (which CDNs and edge servers) to allocate extra bandwidth just as peak demand builds up.

Kadane's Algorithm (2D):
Application: Efficiently finding segments of highest viewer engagement within a geographic heatmap or user-activity timeline.
GIF Source: makeagif.com
For example, if the y-axis represents hours (0–3) and the x-axis represents regions (0–4), Kadane's Algorithm can efficiently find the time-region block with the highest viewer activity. This helps Netflix pinpoint exactly when and where resources need to be scaled up more than usual.
Time Complexity: O(R² × C), where R = number of rows, C = number of columns. Efficient for moderate-sized grids (e.g., 100×100).
This case helps to identify when and where exactly resources should be scaled in order to prevent buffering and optimise user experience.

Python implementation of Kadane's 2D Algorithm:

def kadane_1d(arr):
    max_sum = float('-inf')
    current_sum = 0
    start = 0
    max_start = max_end = 0
    for i, val in enumerate(arr):
        if current_sum <= 0:
            current_sum = val
            start = i
        else:
            current_sum += val
        if current_sum > max_sum:
            max_sum = current_sum
            max_start = start
            max_end = i
    return max_sum, max_start, max_end

def kadane_2d(matrix):
    rows, cols = len(matrix), len(matrix[0])
    max_sum = float('-inf')
    final_left = final_right = final_top = final_bottom = 0

    for top in range(rows):
        temp = [0] * cols
        for bottom in range(top, rows):
            for col in range(cols):
                temp[col] += matrix[bottom][col]
            curr_sum, left, right = kadane_1d(temp)
            if curr_sum > max_sum:
                max_sum = curr_sum
                final_top, final_bottom = top, bottom
                final_left, final_right = left, right
    return max_sum, final_top, final_left, final_bottom, final_right
